PQC — post-quantum asymmetric (ML-KEM/ML-DSA)Hybrid — classical ⊕ PQCQR — quantum-resistant symmetric (AES-256, SHA-384/512…)classical — quantum-broken (RSA/ECC) or sub-strength
On the four classes. PQC is reserved for the new NIST asymmetric standards (FIPS 203 ML-KEM, FIPS 204 ML-DSA) that replace RSA/ECC — which Shor's algorithm breaks outright (classical). QR covers symmetric & hash primitives that stay secure against a quantum adversary at their size: Grover only square-roots symmetric search, so AES-256 keeps ~128-bit security and SHA-384/512/SHA3/KMAC keep their margins — CNSA 2.0 keeps exactly these. So AES-256 is not PQC (that's an asymmetric term) but it is quantum-resistant — it is not "classical". Sub-strength symmetric (AES-128 → ~64-bit under Grover; SHA-256 collision) stays classical.
A second, orthogonal axis — construction & role (the green notes in each row). Two primitives can share a quantum class yet differ structurally: SHA3/SHAKE/KMAC use the Keccak sponge, which is length-extension-immune — so KMAC (SP 800-185) keys a hash directly with no HMAC nesting, and it's design-diverse from SHA-2's Merkle–Damgård (which is length-extendable, hence HMAC). That is not a quantum-status difference — SHA-384/512 and SHA3-384/512 are equally QR. Likewise Argon2id is not "just a KDF": it is a memory-hard passphrase KDF for low-entropy inputs (GPU/ASIC-resistant), a different role from HKDF/KMAC-KDF, which are fast extractors for high-entropy inputs, and stronger than PBKDF2 (iterations only, no memory-hardness). The Class badge is the quantum axis; the green note is the construction/role axis.
