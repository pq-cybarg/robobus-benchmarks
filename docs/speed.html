requirements · speed under each profile

Pick your requirement, see the cost

Security is a spectrum you choose per link, not a mandate — from classical (backwards-compatible with non-PQ peers) through hybrid transition profiles to CNSA 2.0 for National Security Systems. Each profile splits into two cost centres: session setup (the ML-KEM / ML-DSA handshake, once per peer, amortized) and the per-message hot path (AES-256-GCM, every frame). Session cost swings ~130× with assurance; per-message stays flat at line rate — so higher security buys handshake latency, not throughput.

crypto · every primitive · every language · grouped by technique

The crypto matrix

robobus is a post-quantum bus, so its crypto is a whole suite, not one cipher. This is the complete grid — every one of the 26 primitives robobus uses (AEAD ciphers, SHA-2/SHA-3/BLAKE3 hashes, Argon2id/scrypt/PBKDF2/HKDF/KMAC KDFs, ML-KEM + X25519 KEMs, and ML-DSA/SLH-DSA/Falcon + Ed25519 signatures) measured in every one of the 33 language configs, grouped per technique so each bar chart is a coherent apples-to-apples comparison (identical workload, ranked fastest first). Each language uses its native-maximum stack where it has one (stdlib, CryptoKit/JCE, RustCrypto, OpenSSL via its own FFI…); where a language has no native implementation — most of the SHA-3, BLAKE3, PQC and classical-curve cells — it reaches the primitive through one labeled C shim over OpenSSL 3.6.3 + liboqs +  libblake3, so no cell is empty. The post-quantum groups also carry the distinct backends (OpenSSL EVP, liboqs, Go's native crypto/mlkem) for an implementation-quality comparison. Argon2id is known-answer-verified in every language.

ruby runtimes · the ruby family

The same story holds beyond Python — a three-rung runtime ladder on one CPU kernel. transport × language

The full product

Each cell: that language's native client pushing the sealed frame over that transport (frames/s, loopback, pipelined), across the three universal socket transports every language ships natively — UDP, TCP, and Unix-domain sockets (the fastest local socket, no IP stack). These cells are I/O-syscall-bound: every language issues the same send/recv into the same kernel, so within a row throughput converges to that transport's loopback ceiling and small per-language differences (even Python edging C) are run-to-run noise, not language speed. Read it by row, not by cell: the ranking is UDS > TCP > UDP (their kernel paths), and the transport sets the ceiling, not the caller — the opposite of the codec table, where language spans ~1,700×.
