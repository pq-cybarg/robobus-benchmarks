requirements · speed under each profile

Pick your requirement, see the cost

Security is a spectrum you choose per link, not a mandate — from classical (backwards-compatible with non-PQ peers) through hybrid transition profiles to CNSA 2.0 for National Security Systems. Each profile splits into two cost centres: session setup (the ML-KEM / ML-DSA handshake, once per peer, amortized) and the per-message hot path (AES-256-GCM, every frame). Session cost swings ~130× with assurance; per-message stays flat at line rate — so higher security buys handshake latency, not throughput.

python runtimes · CPython · Cython · PyPy+cffi

Same code, four runtimes — pick your stack

robobus's Python surface runs on whatever runtime fits, and each has a native acceleration path. Cython stacks with CPython (seal/open compiled to C in-process); PyPy does not stack with Cython (its cpyext C-API layer is slow) but stacks with native code via cffi — the JIT optimizes straight through the FFI into librobobus. transport × language

The full product

Each cell: that language's native client pushing the sealed frame over that transport (frames/s, loopback, pipelined). These cells are I/O-syscall-bound — every language issues the same send/recv into the same kernel, so throughput converges to the loopback ceiling and small differences (even Python edging C) are run-to-run noise, not language speed. That's the point: the transport sets the ceiling, not the caller — the opposite of the codec table above, where language speed spans ~20,000×.
