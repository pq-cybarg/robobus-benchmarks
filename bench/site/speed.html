Choose every axis, language, KEM, signature, AEAD cipher, hash, keystore KDF, transport, and this composes a full performance specification from the real measurements on this page: session handshake, per-message envelope, transport rate, keystore-unlock cost, and the end-to-end pipeline, alongside the NIST level and standards your combo satisfies. It's the spec sheet you'd hand to a competitor for a head-to-head. Two smart presets: Fastest possible (lowest latency, any security) and Most secure + fast (NIST-5 everywhere, then the fastest option within that level, equivalent NIST classes count as equally secure). transport × language · the full product

Every language, every transport

The complete grid: each of 26 languages driving each of 12 transports with the real 75-byte sealed frame (frames/s, loopback). The three socket transports (UDP/TCP/UDS) each language drives with its own native BSD sockets, or libc sockets via C interop for Fortran/Nelua/Pascal/COBOL/Mojo. The broker & middleware rows (SHM, Serial, CAN, MQTT, AMQP, Kafka, DDS, ZeroCM, LSL, marked ·shim) have no native per-language client, so every language reaches them the same way the crypto matrix reaches a missing primitive: through one labeled C shim (librbxport) wrapping each transport's real C library (libmosquitto, rabbitmq-c, librdkafka, CycloneDDS, libzcm, liblsl, plus an in-C virtual CAN bus and POSIX shm/pty), called through the language's own FFI. Read it by row: the transport work happens in shared C, so within a row languages converge (the FFI overhead is nearly free) and the spread is run-to-run noise on a shared host, not language speed. The transport sets the ceiling, not the caller, the opposite of the codec table (language spans ~1,700×). Colour is log-scaled; the grid spans SHM (~50M f/s, an in-process ring) down to Kafka (~15k, a full broker round-trip). Every cell is filled, all 26 languages × 12 transports, zero blanks. Where a language ships no native socket client (TCP/UDS in the JVM, Node, Lua, Mojo, Octave…) it reaches the socket through the same shim's UDP/TCP/UDS path. Runtime/accelerator variants share their base language's row.

crypto · every primitive · every language · grouped by technique

The crypto matrix

robobus carries a whole crypto suite, not one cipher, so this is the complete grid: every one of the 26 primitives it uses (AEAD ciphers, SHA-2/SHA-3/BLAKE3 hashes, Argon2id/scrypt/PBKDF2/HKDF/KMAC KDFs, the ML-KEM and X25519 KEMs, and the ML-DSA/SLH-DSA/Falcon and Ed25519 signatures) timed in every one of the 33 language configs, grouped per technique so each bar chart is a coherent apples-to-apples comparison (identical workload, ranked fastest first). Pick whichever primitives your requirements call for; the configurator below reports which compliance regime (FIPS, CNSA 2.0, CSfC, NIST level) any given selection satisfies. Each language uses its native-maximum stack where it has one (stdlib, CryptoKit/JCE, RustCrypto, OpenSSL via its own FFI); where a language has no native implementation it reaches the primitive through one labeled C shim over OpenSSL 3.6.3 + liboqs + libblake3, so no cell is empty. The KEM and signature groups also carry the distinct backends (OpenSSL EVP, liboqs, Go's native crypto/mlkem) for an implementation-quality comparison. Argon2id is known-answer-verified in every language.
